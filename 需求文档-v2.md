# 射击成绩AI分析系统 - 需求文档 v2.0

## 更新日期
2026-04-24

## 版本说明
本版本基于与老师对接后的需求调整，主要变更：
1. 新增本地目录读取模式（老师APP抓取图片到指定目录）
2. 保留原有截图上传功能（双模式并存，不冲突）
3. 简化预处理流程（去除靶面识别，老师已提供纯靶面图片）

---

## 一、项目概述

开发一个射击成绩AI分析系统，支持两种图片获取方式：
1. **本地目录模式**：读取老师APP抓取的靶面图片（主模式）
2. **截图上传模式**：Android端截图上传（保留模式）

系统自动对图片进行AI分析，识别弹孔位置、环数和轨迹特征，返回结构化分析结果。

---

## 二、核心需求

### 2.1 双模式图片获取（不冲突）

#### 模式A：本地目录读取（主模式）
- 老师APP将靶面图片保存到指定本地目录
- 系统监控该目录，自动读取新图片
- 支持手动选择目录中的图片进行分析
- 分析完成后可自动归档或删除原图（可配置）

#### 模式B：截图上传（保留模式）
- 基于Android MediaProjection API实现
- 一键截图并上传到服务器
- 不保存本地文件，内存直传
- 与模式A完全独立，互不影响

### 2.2 图片预处理（简化版）

**原复杂流程（已废弃）**：
```
原图 → 裁剪靶纸区域 → 缩放 → CLAHE → 锐化 → 降噪 → JPEG编码
```

**新简化流程**：
```
原图 → 格式统一（JPEG） → 压缩（控制token） → AI分析
```

**简化原因**：
- 老师APP只抓取靶面区域，无需靶面识别
- 无需复杂的裁剪、增强、降噪处理
- 只做简单的格式转换和压缩控制

### 2.3 AI分析功能

保持原有功能不变：
- 识别弹孔（紫色点）位置和环数
- 分析红/蓝/绿三色轨迹特征
- 评估射击技术质量
- 返回结构化JSON结果

### 2.4 管理网站功能

#### 首页Tab调整
- **Tab1 - 实时上传**：保留截图上传模式的实时展示
- **Tab2 - 本地目录**：新增，展示本地目录读取的图片
- **Tab3 - AI分析**：分析结果展示（双模式共用）

#### 本地目录页面功能
- 配置本地目录路径
- 显示目录中的图片列表
- 手动触发分析
- 自动监控新文件（可选）
- 分析状态显示

### 3.2 简化的预处理流程

```javascript
async function simplePreprocess(imagePath) {
  // 1. 读取图片
  const image = await sharp(imagePath);
  
  // 2. 获取尺寸
  const metadata = await image.metadata();
  
  // 3. 只需要压缩控制token消耗
  // 保持原比例，最大边不超过1600px
  const maxDimension = 1600;
  let width = metadata.width;
  let height = metadata.height;
  
  if (width > maxDimension || height > maxDimension) {
    if (width > height) {
      height = Math.round(height * (maxDimension / width));
      width = maxDimension;
    } else {
      width = Math.round(width * (maxDimension / height));
      height = maxDimension;
    }
  }
  
  // 4. 输出JPEG，质量85%
  const processedPath = imagePath.replace(/\.(png|jpg|jpeg|webp)$/i, '_processed.jpg');
  await image
    .resize(width, height, { fit: 'inside' })
    .jpeg({ quality: 85, progressive: true })
    .toFile(processedPath);
  
  return processedPath;
}
```

### 3.3 双模式数据存储

**截图上传模式**：
- 保存到 `backend/screenshots/`
- 记录到 `records.json`
- 保留原有逻辑不变

**本地目录模式**：
- 原图保留在老师指定的目录（不移动）
- 分析结果保存到 `backend/analysis-results/`
- 记录到 `local-records.json`（独立存储，不冲突）

### 3.4 数据隔离设计

```
backend/
├── screenshots/          # 截图上传模式的图片
├── records.json          # 截图上传模式的记录
├── local-analysis/       # 本地目录模式的分析结果
│   ├── results/          # 分析结果JSON
│   └── processed/        # 处理后的图片（可选）
└── local-records.json    # 本地目录模式的记录
```

```javascript
async function analyzeLocalImage(imagePath) {
  // 1. 简单预处理（只压缩，不裁剪增强）
  const processedPath = await simplePreprocess(imagePath);
  
  // 2. 转为Base64

  // 3. 调用AI

  // 4. 保存结果

  return result;
```

## 八、注意事项

1. **权限问题**：确保Node.js进程有权限读取老师APP保存图片的目录
2. **文件占用**：老师APP可能正在写入图片时，需要等待写入完成再读取
3. **路径格式**：Windows和Linux路径格式兼容
4. **存储隔离**：两种模式的记录完全隔离，避免数据混乱
5. **配置持久化**：目录配置保存到配置文件，重启后依然有效
