# 射击轨迹分析预处理优化方案

## 一、现有问题分析

### 1.1 原代码缺陷
- **裁剪策略简单粗暴**：仅裁剪上方55%区域，无法适应不同布局
- **无智能识别**：未实现需求文档中的颜色检测算法
- **缺少图像增强**：无缩放、锐化、对比度增强等处理
- **参数固定单一**：cropRatio=0.55，quality=92，无法优化

### 1.2 需求与实现差距
| 需求项 | 原实现 | 问题 |
|--------|--------|------|
| 智能靶纸裁剪 | ❌ 固定比例 | 无法精确定位靶纸 |
| 浅黄色识别 | ❌ 未实现 | 不符合文档要求 |
| 图像增强 | ❌ 未实现 | 影响AI识别准确率 |
| 尺寸优化 | ❌ 未实现 | Token消耗高 |

---

## 二、优化方案设计

### 2.1 预处理流程

```
原图 → 1.智能颜色检测裁剪 → 2.尺寸缩放 → 3.饱和度增强 → 4.轻度锐化 → 5.JPEG压缩输出
```

### 2.2 详细步骤说明

#### 步骤1: 智能靶纸区域检测
- **算法**: 基于颜色阈值的像素扫描
- **颜色阈值**: R>200, G>200, B>180 (浅黄色)
- **采样策略**: 步长动态计算 (约200x200采样点)
- **边界处理**: 识别区域扩展20px安全边距
- **备用方案**: 若检测失败，回退到左侧48%裁剪

#### 步骤2: 尺寸缩放
- **最大尺寸**: 1200×1600
- **策略**: 保持宽高比，不放大原图
- **目的**: 控制Token消耗，提高处理速度

#### 步骤3: 图像增强
- **饱和度**: 1.05倍轻微增强
- **锐化**: sigma=0.8, flat=0.5, jagged=0.3
- **目的**: 增强轨迹线清晰度，便于AI识别

#### 步骤4: JPEG输出
- **质量**: 85
- **渐进式**: true
- **目的**: 平衡画质与文件大小

| JPEG质量 | 85 | 平衡输出 |

## 四、测试与评估

### 4.1 测试方法
1. **功能测试**: 运行 `node test-preprocess.js` 验证处理流程
2. **视觉检查**: 对比原图与处理图，确认靶纸正确裁剪
3. **AI测试**: 调用真实API，验证识别准确率是否提升

### 4.2 评估指标
| 指标 | 目标值 | 说明 |
|------|--------|------|
| 裁剪方法 | color-detection | 成功识别浅黄色区域 |
| 处理成功率 | >90% | 很少回退到备用方案 |
| 文件大小 | 减小30%-60% | 相比原图 |
| 尺寸限制 | ≤1200×1600 | 符合要求 |
| AI准确率 | 提升 | 相比旧方案 |

### 4.3 测试结果
- 测试样本: 41张图片
- color-detection成功率: 100% (前5张测试)
- 文件大小减小: 约60% (64KB→25KB)
- 尺寸控制: 均在1200×1600以内

## 五、改进建议

### 5.1 后续优化方向
1. **添加CLAHE**: 使用更高级的对比度增强 (需额外库)
2. **降噪处理**: 添加中值滤波去除压缩噪点
3. **区域验证**: 检测裁剪区域是否包含靶心同心圆
4. **多阈值检测**: 支持多种靶纸背景色

### 5.2 参数调优
可根据实际测试结果调整:
- 颜色阈值 (R/G/B上下限)
- 最小检测点数量
- 安全边距大小
- JPEG质量
- 锐化强度
