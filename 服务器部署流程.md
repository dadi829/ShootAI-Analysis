# 服务器部署流程文档

---

## 📋 部署前准备

### 需要准备的服务器
- 操作系统：推荐 Linux（Ubuntu/CentOS）
- 基础环境：Node.js (v18+)

---

## 🔧 部署步骤

### 第一步：上传项目文件
将整个 `ai.analysis` 文件夹上传到服务器，例如：
```
/var/www/ai.analysis/
```

---

### 第二步：配置环境变量

创建环境变量文件 `.env` 在 `backend` 目录下：
```bash
cd /var/www/ai.analysis/backend
nano .env
```

添加以下内容：
```env
# 端口
PORT=3002

# 豆包API配置
DOUBAO_API_KEY=你的豆包API_KEY
DOUBAO_ENDPOINT=你的豆包ENDPOINT
```

保存并退出。

---

### 第三步：安装依赖并启动后端

#### 1. 安装后端依赖
```bash
cd /var/www/ai.analysis/backend
npm install
```

#### 2. 启动后端
```bash
# 方式1：临时启动（开发测试用）
npm start

# 方式2：后台持久运行（推荐，生产环境）
# 先安装 pm2
npm install -g pm2

# 启动服务
pm2 start server.js --name ai-shooting-backend

# 查看状态
pm2 status

# 查看日志
pm2 logs ai-shooting-backend
```

---

### 第四步：构建并部署前端

#### 1. 安装前端依赖
```bash
cd /var/www/ai.analysis
npm install
```

#### 2. 修改后端地址（重要！）
打开 `src/App.tsx`，找到第12行，改成你的服务器地址：
```typescript
const BACKEND_URL = 'http://你的服务器IP:3002';
```
例如：
```typescript
const BACKEND_URL = 'http://192.168.31.175:3002';
```

#### 3. 构建前端
```bash
npm run build
```

#### 4. 部署前端（两种方式）

##### 方式A：使用 Nginx（推荐）
1. 安装 Nginx（如没有）
```bash
# Ubuntu
sudo apt update
sudo apt install nginx

# CentOS
sudo yum install nginx
```

2. 配置 Nginx
创建配置文件 `/etc/nginx/sites-available/ai-shooting`：
```nginx
server {
    listen 80;
    server_name _;

    # 前端
    location / {
        root /var/www/ai.analysis/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 后端API代理
    location /api/ {
        proxy_pass http://localhost:3002/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 图片访问
    location /screenshots/ {
        alias /var/www/ai.analysis/backend/screenshots/;
    }
}
```

3. 启用配置
```bash
sudo ln -s /etc/nginx/sites-available/ai-shooting /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

##### 方式B：使用 Vite 预览（快速测试）
```bash
# 只适合测试，不适合生产
npm run preview -- --host 0.0.0.0 --port 5173
```

## 🌐 访问测试

### 1. 检查后端运行
在浏览器访问：
```
http://你的服务器IP:3002/api/records
```
应该返回空数据或现有数据。

### 2. 检查前端
在浏览器访问：
```
http://你的服务器IP/
```

## 🔒 防火墙设置（如需要）

如果是云服务器，可能需要开放端口：

```bash
# Ubuntu
sudo ufw allow 3002
sudo ufw allow 80

# CentOS
sudo firewall-cmd --permanent --add-port=3002/tcp
sudo firewall-cmd --permanent --add-port=80/tcp
sudo firewall-cmd --reload
```

```

│   ├── server.js
│   ├── screenshots/  # 上传图片存放
│   ├── records.json  # 数据存储
│   └── .env          # 环境变量
├── src/              # 前端源码
├── dist/             # 前端构建产物（部署用）
└── ...
```

---

## ⚠️ 注意事项

1. **保持文件权限正常**：确保运行用户有读写权限
2. **API密钥安全**：不要提交 .env 到git
3. **数据备份**：定期备份 `records.json` 和 `screenshots`
4. **进程守护**：生产环境务必用 pm2 或 systemd 运行后端

- [ ] 前端已构建：`dist/` 目录存在

- [ ] API可访问：浏览器访问 `/api/records` 有返回
- [ ] 图片可访问：上传图片后能显示
- [ ] 分析功能正常：选择图片能返回分析结果
