# Android 屏幕截图上传工具 - 需求文档

## 一、项目概述

开发一个可独立运行、也可被第三方Android App集成的工具模块，核心功能是：用户触发后一键截取当前手机屏幕，不保存任何图片到本地，直接将截图的二进制流上传到指定服务器接口，供后续AI分析使用。

---

## 二、核心需求（必须实现）

### 1. 截图功能
- 基于Android官方 MediaProjection API实现，适配Android 5.0及以上版本
- 截取当前手机屏幕（包含第三方App界面）
- 截图过程不生成本地文件，不写入相册/文件管理器，仅在内存中处理数据

### 2. 权限处理
- 自动申请屏幕录制/截图权限（系统弹窗授权，仅首次触发）
- 权限被用户拒绝时，给出明确提示并终止流程
- 适配Android高版本权限规则（如Android 13+的媒体权限、前台服务权限）

### 3. 上传功能
- 截图后直接将内存中的图片数据（Bitmap/字节流）转为二进制流
- 通过HTTP POST请求上传到指定服务器URL
- 支持配置服务器地址，可设置超时时间
- 上传成功/失败需提供回调（如返回成功状态码、错误信息）

### 4. 无侵入性（适配第三方集成）
- 核心逻辑封装为独立类，对外暴露单一入口方法，无需依赖特定UI
- 可被第三方App直接调用，不强制依赖本工具的Activity界面
- 若需独立运行，提供极简测试UI（仅含"测试截图上传"按钮）

---

## 三、详细功能拆解

### 1. 权限申请模块
- 实现 MediaProjectionManager 授权流程，处理 onActivityResult 回调
- 权限请求逻辑可在Activity中触发，也支持透明Activity模式（无界面跳转感）
- 权限状态缓存，已授权状态下无需重复弹窗

### 2. 截图核心模块
- 通过 ImageReader 获取屏幕帧数据，直接转为Bitmap，不写入本地文件
- 截图完成后立即释放MediaProjection资源，避免内存泄漏
- 支持截图后直接转为字节数组（无需压缩文件），减少内存占用

### 3. 网络上传模块
- 基于OkHttp实现POST请求，支持multipart/form-data格式上传二进制流
- 配置网络请求超时（如15秒），处理无网络、服务器错误等异常场景
- 上传过程支持回调通知（如开始上传、上传进度、上传完成/失败）

### 4. 异常处理模块
- 权限被拒、截图失败、网络异常、服务器返回错误码时，均有对应错误回调
- 不崩溃、不闪退，异常场景下仅终止流程并返回错误信息

---

## 四、非功能需求

### 1. 无本地存储
全程不生成任何图片文件，不访问外部存储权限（Android 10+）

### 2. 低侵入性
核心代码与第三方App业务逻辑解耦，仅需复制核心类即可集成

### 3. 可维护性
代码结构清晰，关键步骤添加注释，核心方法命名规范

### 4. 兼容性
适配主流Android版本（API 21 - API 34），处理不同版本的权限差异

---

## 五、交付要求

### 1. 完整项目
完整的Android Studio项目代码，包含所有依赖配置

### 2. 核心封装类
核心功能封装为独立类（如 ScreenCaptureUploadHelper.java），对外暴露以下方法：

```java
// 入口方法：触发截图并上传
public static void startCaptureAndUpload(Activity activity, String serverUrl, UploadCallback callback);

// 上传结果回调接口
public interface UploadCallback {
    void onSuccess(String response);
    void onFailed(int errorCode, String errorMsg);
}
```

### 3. 测试Activity
提供极简测试Activity（含一个按钮，点击触发截图上传，便于验证功能）

### 4. 集成说明
输出一份简短的集成说明（说明第三方App如何调用核心方法）

---

## 六、补充说明

- 无需实现悬浮窗功能，优先保证第三方App直接调用的集成方式
- 无需处理图片压缩以外的复杂处理，仅需保证图片数据完整上传
- 不添加任何额外UI元素，若需独立运行，仅保留必要的授权提示
