# 气步枪打靶数据分析系统 - 需求文档

## 1. 项目概述

### 1.1 项目背景
本项目是一个基于AI的气步枪射击数据分析系统，通过上传射击训练截图，利用AI进行智能分析，为射击运动员提供数据洞察和改进建议。

### 1.2 项目目标
- 提供图片上传功能，支持射击训练系统截图的选择和上传
- 集成AI视觉分析能力，自动提取图片中的射击数据
- 呈现详细的数据分析结果，帮助用户了解射击表现

---

## 2. 功能需求

### 2.1 图片选择与上传功能
- **功能描述**：用户可以从本地设备选择射击训练系统的截图
- **功能细节**：
  - 支持常见图片格式（PNG, JPG, JPEG）
  - 提供文件选择对话框
  - 支持拖拽上传
  - 上传前显示图片预览
  - 文件大小限制（建议<10MB）

### 2.2 AI分析功能
- **功能描述**：调用AI接口分析上传的图片，提取射击数据
- **数据提取内容**：
  - 序号、方向、分数、总分、时间、间隔
  - 平均分数计算
  - 最高分/最低分识别
  - 射击趋势分析
  - 靶心分布分析
  - 射击间隔统计

### 2.3 分析结果展示
- **功能描述**：将AI分析结果以可视化方式呈现
- **展示内容**：
  - 数据表格（原始数据）
  - 分数统计图表（柱状图/折线图）
  - 关键指标卡片（平均分、最高分、最低分、总分等）
  - 射击趋势分析图
  - AI建议和改进意见

### 2.4 历史数据功能
- **功能描述**：保存和管理历史分析记录，支持数据查询、对比和趋势分析
- **数据收集范围**：
  - 所有AI分析结果（原始数据、统计数据、趋势分析、AI建议）
  - 上传的原始图片（可选，用于后续参考）
  - 分析时间戳
  - 用户添加的备注信息
  - 训练类型和环境信息

- **存储格式**：
  - 前端存储：使用浏览器 localStorage 或 IndexedDB（适合本地使用）
  - 后端存储（可选）：使用数据库存储，支持多设备同步
  - 数据结构：
    ```typescript
    interface HistoryRecord {
      id: string;              // 唯一标识
      timestamp: number;       // 分析时间戳
      analysisResult: AnalysisResult;  // 分析结果
      imageUrl?: string;       // 图片存储路径（可选）
      notes?: string;          // 用户备注
      trainingType?: string;   // 训练类型
      environment?: string;    // 环境信息
      tags?: string[];         // 标签
    }
    ```

- **检索方法**：
  - 按时间范围查询（日/周/月/年）
  - 按训练类型筛选
  - 按分数范围筛选
  - 按标签筛选
  - 关键词搜索（备注内容）

- **显示规格**：
  - 历史记录列表（表格形式）
  - 每条记录显示关键信息：时间、总分、平均分、训练类型
  - 点击记录可查看详细分析结果
  - 支持多选记录进行对比分析
  - 趋势图表（长期数据变化）
  - 数据导出功能（CSV/Excel格式）

- **更新频率**：
  - 实时保存：每次分析完成后自动保存
  - 批量导入：支持从文件导入历史数据
  - 手动编辑：支持修改备注和标签

- **访问权限**：
  - 本地存储：仅当前用户可访问
  - 云端存储：基于用户账户权限
  - 数据隐私：所有数据加密存储
  - 导出权限：用户可导出自己的历史数据

## 3. 技术栈建议

### 3.1 前端技术方案
- **框架选择**：
  - 方案A：React + TypeScript + Ant Design（推荐，企业级UI组件库）
  - 方案B：Vue 3 + TypeScript + Element Plus
  - 方案C：Next.js（全栈React框架）

- **图表库**：
  - ECharts（功能强大，适合数据可视化）
  - 或 Recharts（React生态，轻量级）

### 3.2 后端技术方案（如需要）
- **语言**：Python / Node.js / Java
- **框架**：Flask / FastAPI / Express / Spring Boot
- **AI接口**：
  - 方案A：使用OpenAI GPT-4V / Claude 3 Vision（图片理解能力强）
  - 方案B：使用国内大模型（如通义千问、文心一言等的视觉能力）
  - 方案C：自研OCR + 数据分析模型

### 3.3 部署方案
- 前端：Vercel / Netlify / 静态托管
- 后端：云服务器 / Serverless函数

## 5. 数据结构设计

### 5.1 射击数据结构
```typescript
interface ShootingData {
  serialNumber: number;      // 序号
  direction: string;         // 方向
  score: number;             // 分数
  totalScore: number;        // 总分
  time: number;              // 时间
  interval: number;          // 间隔
}
```

### 5.2 分析结果结构
```typescript
interface AnalysisResult {
  rawData: ShootingData[];           // 原始数据
  statistics: {
    averageScore: number;            // 平均分
    highestScore: number;            // 最高分
    lowestScore: number;             // 最低分
    totalScore: number;              // 总分
    totalShots: number;              // 总射击次数
    averageInterval: number;         // 平均间隔
  };
  trend: {
    isImproving: boolean;            // 是否有提升趋势
    description: string;             // 趋势描述
  };
  aiSuggestions: string[];           // AI建议
}
```

## 6. AI分析功能详细设计

### 6.1 分析维度
1. **成绩稳定性分析**：分数的方差、标准差
2. **趋势分析**：分数随时间的变化趋势
3. **射击节奏分析**：间隔时间的规律性
4. **方向分布**：各方向的射击次数和平均分
5. **靶心分布**：弹着点的集中程度

### 6.2 AI提示词设计思路
- 识别图片类型（射击训练系统截图）
- 提取表格数据（序号、方向、分数、总分、时间、间隔）
- 计算统计指标
- 生成改进建议
