**内容**：

**视觉设计**：

### 第3页：整体技术方案
**内容**：
- **系统架构图**（三层架构）：
  ```
  感知层(硬件授权) → 传输层(蓝牙BLE 5.0) → 应用层(端云协同)
      ↓
  数据采集 → 图像预处理 → AI分析 → 可视化展示
  ```
- **技术路线**：基于现有IoT基础设施的增量式智能化升级路径
- **设计原则**：轻量化、实时性、可靠性、可扩展性

**视觉设计**：

- 数据流向箭头标注

### 第4页：创新点1+2（合并布局）
**布局**：左右分栏，每栏包含创新点标题、技术原理、创新价值、量化指标

#### 左栏：创新点1 - 领域知识增强的Prompt工程方案
**技术原理**：
- 多模态输入约束：构建包含视觉语义映射的System Prompt
- 输出结构化控制：通过JSON Schema约束专业输出规范
- 参数优化策略：temperature=0.25确保输出稳定性

**创新价值**：
- 解决通用大模型专业输出不可控的技术难题
- 实现从"黑盒推理"到"可控诊断"的技术突破
- 为专业运动训练提供可复用的Prompt工程范式

**量化指标**：AI识别准确率99.3%，结构化输出成功率95%，响应时间0.8秒

#### 右栏：创新点2 - 存量IoT设备的零边际成本AI升级方案
**技术原理**：
- 硬件功能抽象层：建立传感器数据与AI分析的标准化接口
- 软件定义功能：通过纯软件更新为固定硬件注入动态AI能力
- 增量部署策略：基于现有基础设施，避免重复投资

**创新价值**：
- 开创"硬件不变、软件增值"的IoT设备智能化新路径
- 解决专用AI设备部署成本高、周期长的行业痛点
- 为传统制造业数字化转型提供轻量化升级范式

**量化指标**：部署成本降低90%，升级周期从月级降至小时级

### 第5页：创新点3+4（合并布局）

#### 左栏：创新点3 - 轻量化端云协同计算架构
**技术原理**：
- 计算任务动态分配：端侧重实时预处理，云侧重智能分析
- 数据流优化策略：自适应图像压缩，减少94%传输数据量
- 容错恢复机制：多级重试策略和JSON解析容错

**创新价值**：
- 突破传统端侧计算能力有限与云端延迟高的技术瓶颈
- 实现计算效率与成本控制的帕累托最优
- 为边缘AI应用提供可扩展的架构参考

**量化指标**：端到端延迟0.8秒，月均服务成本<50元，系统可用性99.9%

#### 右栏：创新点4 - 多模态时序数据融合分析框架
**技术原理**：
- 异构数据对齐算法：基于时钟漂移补偿的时序同步机制
- 特征级融合策略：在特征提取层面实现多源数据互补增强
- 决策级诊断模型：构建基于多维度特征的射击动作评估体系

**创新价值**：
- 解决多源异构传感器数据融合的技术难题
- 实现从单一数据维度到多模态综合诊断的技术跨越
- 为运动科学量化分析提供新的方法论

**量化指标**：时序同步精度±3ms，击发点识别精度±15ms

### 第6页：核心技术实现
**内容**：
- **关键技术流程图**：

  图像输入 → Sharp预处理 → 豆包API → JSON解析 → 环数计算

- **算法原理**：
  - 基于欧几里得距离的环数判定算法

- **技术实现细节**：自适应压缩、JSON容错解析、主线程回调机制

**视觉设计**：

- **团队能力类**：学生贡献占比、知识产权布局
