# 设计思路

## 1. 设计背景

射击训练领域已普遍部署物联网采集设备，包括光电轨迹捕捉系统（100Hz采样）和压力传感系统（1000Hz采样），能够实时记录瞄准轨迹与扳机压力数据。然而，这些设备长期停留在"数据采集+本地显示"阶段，面临"有数据、无智能"的核心矛盾：

- **数据未充分利用**：轨迹与压力数据仅做原始曲线展示，缺乏AI智能诊断能力，动作问题定位依赖教练经验定性判断
- **判环效率低下**：靶面环数判定依赖人工肉眼判读，单张靶面耗时3-5分钟，误差率5-10%
- **数据无法沉淀**：训练记录分散存储于本地设备，无法跨场次追踪分析，更无法支撑长期训练规划
- **军警合规受限**：进口高端系统（如Scatt）数据存储于海外服务器，不符合《数据安全法》第三十一条关于关键数据境内存储的要求；国产军警专用系统虽合规，但价格10万元以上、部署复杂、无AI诊断能力

底层硬件产品已上市多年，服务多家军警训练单位及射击俱乐部，具备成熟的市场渠道与客户基础。在此基础上，如何通过软件层面的AI能力注入，实现现有IoT设备的智能化升级，成为本项目的核心出发点。

## 2. 核心目标

本项目旨在构建一套基于端云AI协同的射击训练智能分析系统，在**不改造现有硬件**的前提下，通过APP软件升级实现以下目标：

| 目标维度 | 具体目标 | 量化指标 |
|:---|:---|:---|
| **功能目标** | 靶面AI自动判环、轨迹可视化诊断、压力曲线分析、智能训练报告 | 响应时间≤1秒，AI识别准确率≥99% |
| **性能目标** | 多源数据时序精准对齐、弱网环境稳定运行 | 时序同步精度±5ms，离线模式切换成功率100% |
| **合规目标** | 全国产化数据存储，支持本地化部署 | 符合《数据安全法》，满足军警涉密要求 |
| **商业目标** | 现有客户零硬件改造成本获得AI能力 | 软件增值模式，即装即用 |

## 3. 关键问题分析

实现上述目标，需要解决三个核心技术问题：

### 3.1 问题一：靶面图像传输效率与AI识别精度的矛盾

**问题描述**：原始靶面图像分辨率通常为1920×1080，直接上传导致传输慢（4G网络下单张上传需5-8秒）、AI处理延迟大，严重影响用户体验。

**问题拆解**：
- 压缩率过高 → 靶面边缘模糊 → AI识别准确率下降
- 压缩率过低 → 传输时间长 → 响应延迟超标
- 通用压缩算法（如微信图片压缩）采用固定参数，不区分场景，无法兼顾效率与精度

### 3.2 问题二：多源异构数据的时序精准对齐

**问题描述**：轨迹数据100Hz、压力数据1000Hz，两个硬件采用独立晶振时钟，存在时钟漂移。单次训练持续1-2小时，累积误差可达±50ms，而击发瞬间仅持续100ms，误差将导致击发点定位失败。

**问题拆解**：
- 通用线性插值仅做简单时间对齐，无法解决硬件时钟漂移
- 长时间训练后累积误差增大，同步精度衰减
- 击发瞬间需要±5ms级精度，通用方案无法满足

### 3.3 问题三：军警赛道的数据合规与部署便捷性

**问题描述**：军警训练单位对数据安全有强制性要求，进口系统数据出境不合规，国产专用系统部署复杂且封闭。如何在保证AI能力的同时，满足国产化合规与轻量化部署？

**问题拆解**：
- AI大模型通常依赖云端API，数据需上传至第三方服务器
- 军警涉密场景要求数据不出境、甚至不出本地
- 专用AI服务器部署成本高、维护复杂，与"轻量化"目标矛盾

## 4. 解决方案构想

针对上述三个问题，本项目采用"端云协同轻量化架构"，通过端侧预处理降低传输负载、自研时序算法保障对齐精度、国产化云服务体系满足合规要求。

### 4.1 问题一解决方案：自适应图像预处理算法

**设计思路**：针对射击靶面"高对比度黑白靶纸、标准环形结构"的特点，设计场景专属压缩策略。

**核心策略**：
- **全局压缩**：等比例缩放至720px，JPEG质量60%，压缩率97%
- **局部保真**：针对靶面中心10环区域（直径仅0.5mm）采用局部保真策略，确保关键区域压缩后不损失识别精度
- **动态评估**：通过分析靶面边缘清晰度动态调整压缩参数，平衡体积与精度

**预期效果**：传输时间减少94%，AI识别准确率保持99%+。

### 4.2 问题二解决方案：基于时钟漂移补偿的时序同步算法

**设计思路**：针对射击训练"单次击发时间短（2-3秒）、连续训练时间长（1-2小时）"的特点，设计分段校时策略。

**核心策略**：
- **实时漂移补偿**：监测两个硬件时钟的漂移率差异，动态修正时间戳偏差
- **分段校时**：每次击发前重新校时，消除累积漂移；击发瞬间采用高频插值（10kHz等效）精确定位
- **压力触发校验**：结合压力突变检测与加速度阈值判断，双重定位击发瞬间

**预期效果**：同步精度稳定在±5ms，优于通用方案（±10-20ms）。

### 4.3 问题三解决方案：全国产化端云协同架构

**设计思路**：采用"端侧重实时、云侧重智能"的分工原则，端侧完成数据预处理与本地缓存，云端完成AI推理与数据持久化，同时支持本地化部署模式。

**核心策略**：
- **国产化云服务**：采用阿里云Serverless架构（函数计算+API网关+OSS），数据全链路境内存储
- **本地部署支持**：关键模块支持私有化部署，满足军警涉密场景"数据不出本地"要求
- **弱网自适应**：SQLite本地缓存+断点续传，弱网环境下端侧可独立运行，网络恢复后自动同步

## 5. 技术路线选择依据

### 5.1 为什么选择端云协同而非纯边缘方案？

纯边缘方案（手机端运行AI模型）虽可规避网络依赖，但面临以下限制：
- 手机算力有限，大模型推理延迟高（>3秒），无法满足"≤1秒响应"目标
- 模型体积大（>1GB），占用手机存储，影响用户体验
- 模型更新需重新发布APP，迭代周期长

端云协同方案将AI推理放在云端，端侧仅做预处理与展示，兼顾响应速度与模型迭代灵活性。

### 5.2 为什么选择端云协同而非纯云端方案？

纯云端方案（所有数据实时上传）虽可简化端侧开发，但面临以下限制：
- 弱网环境下无法使用，训练连续性无法保障
- 原始数据全量上传，流量消耗大，用户体验差
- 实时可视化依赖网络往返，延迟高

端云协同方案在端侧完成预处理与本地缓存，弱网环境下可独立运行，保障训练连续性。

### 5.3 为什么选择自研CV算法+大模型API融合方案？

纯自研CV方案虽可控，但开发周期长、泛化能力有限；纯API方案虽快速，但黑盒处理、无法优化。融合方案以自研CV算法为主（覆盖95%常规场景），大模型API为辅（仅用于疑难场景校验），兼顾效率与精度。

## 6. 创新点凝练

本项目的创新并非追求单一技术的突破，而是针对射击训练场景的**系统化融合创新**：

| 创新维度 | 创新内容 | 与通用方案的区别 |
|:---|:---|:---|
| **设计思路** | 基于现有IoT设备的AI软件增值 | 无需改造硬件，保护既有投资；竞品需全套专用设备 |
| **算法创新** | 时钟漂移补偿+分段校时算法 | 通用线性插值无法解决硬件漂移；本方案±5ms精度 |
| **场景优化** | 射击场景专属图像压缩与AI诊断 | 通用压缩不区分场景；本方案局部保真策略 |
| **合规创新** | 全国产化存储+本地部署支持 | 进口竞品数据存海外；本方案符合《数据安全法》 |

## 7. 设计思路总结

本项目的设计思路遵循"问题导向、场景驱动、融合创新"的原则：从射击训练物联网设备"有数据、无智能"的真实痛点出发，通过端云协同架构实现现有设备的智能化升级，以自研核心算法解决时序对齐与图像压缩的技术难题，以国产化合规设计满足军警赛道强制要求。最终形成一套"即装即用、零硬件改造成本、AI智能诊断"的射击训练分析解决方案，填补进口系统（不合规）与国产专用系统（无AI、部署重）之间的市场空白。
