# 📱 Android真机测试指南

---

## 第一步：杀掉占用端口的进程

**问题：** 3002端口被占用了

**解决方法：**
1. **双击运行**：`d:\ai.analysis\kill-port-3002.bat`
2. 或者手动操作（如果上面不行）：
   ```bash
   # 打开cmd
   netstat -ano | findstr :3002
   # 记下PID，比如 12345
   taskkill /PID 12345 /F
   ```

---

## 第二步：启动后端

```bash
cd d:\ai.analysis\backend
npm start
```

看到这个就成功了：
```
🚀 服务已启动！
📍 访问地址: http://localhost:3002
```

---

## 第三步：获取你电脑的IP地址

1. **打开cmd**
2. **输入：** `ipconfig`
3. **找到你的IPv4地址**，类似这样：
   ```
   IPv4 地址 . . . . . . . . . . . . : 192.168.1.100
   ```
4. **记下这个IP**（每个人的不一样）

---

## 第四步：修改App配置

1. **打开文件：** `d:\ai.analysis\mobile-app\src\config.ts`
2. **修改这一行：**
   ```typescript
   baseUrl: 'http://192.168.1.100:3002',  // 把IP改成你自己的！
   ```
3. **保存文件**

---

## 第五步：配置手机

### 5.1 开启开发者选项
- 打开手机 **设置** → **关于手机**
- 连续点击 **版本号** 7次
- 提示"您已处于开发者模式！"

### 5.2 开启USB调试
- 返回 **设置** → 找到 **开发者选项**
- 打开 **USB调试** 开关

### 5.3 连接手机到电脑
- 用数据线连接手机和电脑
- 手机上弹出"允许USB调试吗？" → 点击 **允许**
- 手机连接模式选择 **文件传输（MTP）**

### 5.4 确认同一WiFi
- 手机和电脑连 **同一个WiFi**

---

## 第六步：构建并安装App到手机

### 6.1 构建App前端
```bash
cd d:\ai.analysis\mobile-app
npm run build
```

### 6.2 同步到Android
```bash
npm run sync
```

### 6.3 在Android Studio中运行
1. 打开 Android Studio
2. 打开项目：`d:\ai.analysis\mobile-app\android`
3. 等待Gradle同步完成（可能需要几分钟）
4. 顶部设备选择栏选择你的手机
5. 点击绿色的运行按钮（▶️）
6. 等待App安装到手机

---

## 第七步：测试！

### 7.1 在手机上测试
1. 打开刚安装的App
2. 点击右下角相机按钮 📷
3. 选择 **从相册选择** 或 **拍照**
4. 选择AI模型：**模拟数据（测试用）**
5. 点击 **开始分析**
6. 等待2秒，应该看到分析结果！

### 7.2 验证是否成功
- **检查手机App：** 是否显示分析结果
- **检查电脑后端：** `backend/screenshots` 目录有没有新上传的图片
- **检查后端日志：** cmd窗口应该有请求日志

## 常见问题

### Q1：手机连不上电脑后端
**A：**
1. 确认手机和电脑在同一WiFi
2. 关闭电脑防火墙（临时测试）
3. 确认IP地址正确

### Q2：Android Studio找不到手机
**A：**
1. 换个USB口
2. 换根数据线（有些只能充电不能传数据）
3. 手机上重新允许USB调试

### Q3：App安装失败
**A：**
1. 卸载手机上旧版本的App
2. 重新运行
