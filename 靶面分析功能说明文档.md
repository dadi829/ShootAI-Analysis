# 靶面分析功能说明文档

## 一、当前APP已实现的功能

### 1.1 功能概述
气步枪打靶数据分析APP，用于上传靶面图片并进行AI分析，识别弹孔位置、环数，并提供技术评价。

### 1.2 核心功能

#### 功能1：服务器地址配置
- **界面元素**：文本输入框
- **默认值**：`http://192.168.31.175:3002`
- **作用**：配置后端API服务器地址
- **部署后**：修改为服务器公网地址

#### 功能2：文件夹选择与持久化
- **界面元素**："📁 选择文件夹"按钮
- **操作流程**：
  1. 首次点击弹出文件管理器
  2. 用户选择目标文件夹（如 `shoot.test`）
  3. 系统授予权限并自动保存
  4. 下次打开APP无需重复选择
- **技术实现**：使用 Storage Access Framework (SAF)

#### 功能3：自动获取最新图片
- **触发条件**：点击"开始分析"按钮
- **处理逻辑**：
  1. 扫描已选文件夹
  2. 筛选图片文件（.jpg/.jpeg/.png/.webp）
  3. 根据文件修改时间找到最新的图片
  4. 自动复制到APP缓存目录

#### 功能4：图片上传
- **API接口**：`POST /upload`
- **Content-Type**：`multipart/form-data`
- **上传参数**：
  - `image`：图片文件
- **返回数据**：
  ```json
  {
    "success": true,
    "record": {
      "id": "记录ID",
      "filename": "保存的文件名",
      "url": "/screenshots/xxx.jpg"
    }
  }
  ```

#### 功能5：AI分析调用
- **API接口**：`POST /api/analyze/trajectory`
- **Content-Type**：`application/json`
- **请求参数**：
  ```json
  {
    "recordId": "上传返回的记录ID"
  }
  ```
- **返回数据**：
  ```json
  {
    "success": true,
    "analysis": {
      "shot": {
        "ring": 9,
        "position": {
          "x": -29.65,
          "y": -17.16
        }
      },
      "summary": {
        "stabilityRating": "B",
        "triggerControlRating": "A",
        "followThroughRating": "A",
        "mainIssue": "Pre - shot aiming trajectory is unstable (with fluctuations)",
        "suggestion": "Practice steady aiming methods to reduce pre - shot movement and enhance aiming stability."
      }
    }
  }
  ```

#### 功能6：分析结果展示
- **展示内容**：
  - 环数
  - 弹孔位置坐标
  - 稳定性评级
  - 扳机控制评级
  - 跟进质量评级
  - 主要问题
  - 改进建议
- **显示位置**：页面底部白色文本区域

### 1.3 完整操作流程

```
用户打开APP
    ↓
确认服务器地址（首次需配置）
    ↓
点击"📁 选择文件夹"（首次需选择）
    ↓
在文件管理器中选择目标文件夹
    ↓
点击"开始分析"
    ↓
系统自动获取文件夹最新图片
    ↓
上传图片到后端
    ↓
调用AI分析
    ↓
显示分析结果
```

## 二、给老师的APP集成方案

### 2.1 集成代码文件

老师只需要一个文件：`TargetImageUploadHelper.java`

这个文件包含全部功能：
- ✅ 配置服务器地址
- ✅ 配置图片路径
- ✅ 上传图片
- ✅ 调用AI分析
- ✅ 回调返回结果

### 2.2 集成接口说明

#### 接口1：初始化与配置

```java
// 创建上传助手
TargetImageUploadHelper uploadHelper = new TargetImageUploadHelper(context);

// 配置服务器地址
uploadHelper.setServerUrl("http://your-server.com:3002");
```

老师的APP已经有图片，可以直接使用：


## 三、具体问题解答

### 问题1：后续修改提示词是否需要重新集成？

**答案：不需要！**

**原因：**
- 提示词修改在**后端代码**中进行
- Android APP只是调用后端API，不包含提示词逻辑
- 只要后端API返回数据的格式保持不变，无需重新集成

**提示词位置**：`backend/server.js` 中的 `buildSystemPrompt()` 函数

**后端部署后**：直接修改后端代码，重启服务即可生效。

---

### 问题2：能否实现一次点击自动上传并返回结果？

**答案：完全可以！**

**已实现的方案：**

```java
// 老师APP只需调用一次
uploadHelper.setTargetImagePath("/sdcard/shoot.test/latest.jpg");

uploadHelper.uploadAndAnalyze(callback);  // 一次调用，自动完成全部！
```

**在老师APP的按钮点击中：**

```java
public void onAnalyzeButtonClick(View view) {
    // 老师的APP自己保存图片到指定路径
    // ... 老师已有这个功能
    
    // 调用我们的接口
    uploadHelper.setTargetImagePath("/sdcard/shoot.test/latest.jpg");
    uploadHelper.uploadAndAnalyze(new TargetImageUploadHelper.UploadCallback() {
        @Override
        public void onSuccess(JSONObject analysisResult) {
            // 在老师的APP上显示结果
            showAnalysisResult(analysisResult);
        }
        
        @Override
        public void onFailed(String errorMsg) {
            showError(errorMsg);
        }
    });
}
```

**完整流程（一次点击）：**

```
老师点击"分析"按钮
    ↓
老师APP保存图片到指定文件夹
    ↓
老师APP调用我们的接口
    ↓
接口自动上传图片
    ↓
接口自动调用AI分析
    ↓
接口返回结果
    ↓
老师APP显示结果
```

### 问题3：AI分析结果能否在老师的APP上同样正常显示？

**答案：完全可以！**

**原因：**
- 我们的APP和老师的APP都处理**相同的JSON数据**
- 显示方式完全由前端控制，数据格式保持一致

**老师APP显示结果的代码示例：**

```java
private void showAnalysisResult(JSONObject result) {
    try {
        // 获取弹孔信息
        JSONObject shot = result.getJSONObject("shot");
        int ring = shot.getInt("ring");
        JSONObject pos = shot.getJSONObject("position");
        double x = pos.getDouble("x");
        double y = pos.getDouble("y");
        
        // 获取技术评价
        JSONObject summary = result.getJSONObject("summary");
        String stability = summary.getString("stabilityRating");
        String trigger = summary.getString("triggerControlRating");
        String follow = summary.getString("followThroughRating");
        String mainIssue = summary.getString("mainIssue");
        String suggestion = summary.getString("suggestion");
        
        // 在老师的APP上显示（示例）
        TextView ringText = findViewById(R.id.ring_text);
        ringText.setText("环数: " + ring);
        
        // ... 其他字段显示
    } catch (Exception e) {
        e.printStackTrace();
    }
}
```

**数据格式保证：**
- 只要后端API返回的JSON字段不变，显示方式完全相同
- 可以根据老师的需求定制显示样式，数据逻辑不需要改

## 四、后续功能迭代建议

### 4.1 迭代时的注意事项

1. **后端修改**（如提示词优化）：
   - ✅ 不需要重新集成
   - ✅ 只需部署更新后的后端

2. **Android功能新增**（如文件选择方式修改）：
   - ⚠️ 需要重新集成
   - ⚠️ 更新 `TargetImageUploadHelper.java`

3. **API返回字段增加**：
   - ✅ 向后兼容，新增字段不影响现有功能
   - ✅ 老师需要时再读取新字段

### 4.2 部署后的配置

1. **服务器地址**：
   - 测试时用局域网IP：`http://192.168.31.175:3002`
   - 部署后用公网地址：`https://your-domain.com/api`

2. **后端服务**：
   - 需要在公网服务器部署
   - 配置防火墙开放3002端口
   - 推荐使用 Nginx 做反向代理和HTTPS

```xml

<!-- 如果需要访问本地文件 -->

```

在回调中解析JSON，用老师的UI显示。
