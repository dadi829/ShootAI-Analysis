# 🧪 测试指南

---

## 一、环境准备

### 1.1 安装依赖

```bash
# 安装后端依赖
cd d:\ai.analysis\backend
npm install

# 安装前端依赖（Web版本）
cd d:\ai.analysis
npm install

# 安装移动端依赖
cd d:\ai.analysis\mobile-app
npm install
```

### 1.2 配置后端（可选）

在 `d:\ai.analysis\backend` 目录下创建 `.env` 文件（如果需要真实AI模型）：

```env
# OpenAI API Key（可选）
OPENAI_API_KEY=your-openai-api-key

# 通义千问 API Key（可选）
TONGYI_API_KEY=your-tongyi-api-key

# 豆包 API Key（可选）
DOUBAO_API_KEY=your-doubao-api-key
```

**注意：** 不配置API Key也可以使用 `mock` 模式测试！

---

## 二、启动后端

### 2.1 启动后端服务

```bash
cd d:\ai.analysis\backend
npm start
```

看到以下输出表示启动成功：

```
🚀 服务已启动！
📍 访问地址: http://localhost:3002
📁 截图保存目录: ...\backend\screenshots
```

### 2.2 测试后端是否正常

在浏览器访问：`http://localhost:3002/api/health`

应该看到：

```json
{
  "status": "ok",
  "message": "服务运行正常"
}
```

## 三、测试Web版本

### 3.1 启动Web前端

打开一个新的终端：

```bash
cd d:\ai.analysis
npm run dev
```

### 3.2 测试分析功能

1. 在浏览器打开：`http://localhost:5173`
2. 选择一张射击训练的截图（或使用 `backend/screenshots` 目录下的示例图片）
3. 选择AI模型（建议先用 `模拟数据` 测试）
4. 点击 `同步并分析`
5. 等待分析完成（mock模式2秒）
6. 查看分析结果

---

## 四、测试Android App

### 4.1 构建并同步

```bash
cd d:\ai.analysis\mobile-app

# 构建前端
npm run build

# 同步到Android
npm run sync
```

### 4.2 在Android Studio中运行

1. 打开Android Studio
2. 打开项目：`d:\ai.analysis\mobile-app\android`
3. 等待Gradle同步完成
4. 连接Android设备或启动模拟器
5. 点击运行按钮

### 4.3 测试App功能

1. 在App中点击右下角相机按钮
2. 选择 `从相册选择` 或 `拍照`
3. 选择AI模型（建议先用 `模拟数据`）
4. 点击 `开始分析`
5. 查看分析结果

**注意：**
- Android模拟器访问本机后端请使用 `http://10.0.2.2:3002`（代码已配置好）
- 真机测试需要把后端部署到局域网或云服务器

## 五、测试API（给老师集成前）

### 5.1 使用curl测试

```bash
# Windows PowerShell
curl -X POST http://localhost:3002/api/analyze `
  -F "screenshot=@d:\ai.analysis\backend\screenshots\screenshot_1776858957085.jpg" `
  -F "model=mock"
```

### 5.2 使用Postman测试

1. 创建新的 POST 请求
2. URL: `http://localhost:3002/api/analyze`
3. Body → form-data
4. Key: `screenshot`, 类型选择 File, Value: 选择一张截图
5. Key: `model`, Value: `mock`
6. 点击 Send
7. 查看响应

### 5.3 预期响应

```json
{
  "success": true,
  "url": "http://localhost:3002/screenshots/screenshot_xxx_processed.jpg",
  "analysis": "---\n### 一、轨迹特征分析\n...\n---",
  "filename": "screenshot_xxx_processed.jpg"
}
```

## 六、常见问题

### Q1: 后端启动失败

**A:** 检查端口3002是否被占用，可以修改 `backend/server.js` 中的端口号

### Q2: 提示 "无法连接后端"

**A:** 确保后端服务已启动，检查URL是否正确

### Q3: AI分析超时

**A:** mock模式应该2秒返回，真实AI模型可能需要5-50秒，请耐心等待

### Q4: Android App无法连接后端

**A:** 
- 模拟器使用 `10.0.2.2` 代替 `localhost`
- 真机需要在同一局域网，使用电脑的IP地址
- 确保没有防火墙阻止
