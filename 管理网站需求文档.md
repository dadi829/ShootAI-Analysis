# 射击成绩 AI 分析 - 管理网站需求文档

## 一、项目概述

开发一个 Web 管理界面，用于管理和查看来自 Android 端上传的射击成绩截图，展示 AI 分析结果，支持历史记录查询和统计。

---

## 二、核心需求（必须实现）

### 1. 首页 - 实时上传展示
- 实时展示最新上传的截图和分析结果
- 列表形式展示，包含：缩略图、上传时间、分析状态
- 点击查看详情

### 2. 分析详情页
- 展示完整截图
- 展示 AI 分析结果（文本形式）
- 展示标注后的分析图片
- 提供重新分析按钮

### 3. 历史记录页
- 分页展示所有历史记录
- 支持按日期范围筛选
- 支持搜索（按文件名或日期）
- 支持删除单条或批量删除

### 4. 统计页面
- 总上传次数统计
- 每日/每周上传趋势图
- 分析成功率统计

---

## 三、详细功能拆解

### 1. 后端 API（复用现有）
- 保持 `/upload` 接口不变
- 新增 `/api/records` - 获取记录列表
- 新增 `/api/records/:id` - 获取单条记录详情
- 新增 `/api/records/:id/reanalyze` - 重新分析
- 新增 `/api/records/:id` - 删除记录
- 新增 `/api/stats` - 获取统计数据

### 2. 前端页面
- **首页（Dashboard）**：最新记录 + 快速统计卡片
- **历史记录（History）**：表格 + 筛选 + 搜索
- **详情页（Detail）**：大图 + 分析结果
- **统计页（Stats）**：图表展示

### 3. 数据存储
- 每条记录保存：原始图、分析图、分析文本、时间戳
- 使用本地 JSON 文件或轻量级数据库（如 SQLite）

---

## 四、技术选型建议

- 前端：React + Vite（现有结构可复用）
- 后端：Node.js + Express（现有）
- UI 组件库：Ant Design 或 Element Plus（美观、组件丰富）
- 图表：ECharts 或 Recharts

---
