# 射击成绩 - 管理网站需求文档（简化版）

## 一、项目概述

开发一个 Web 管理界面，用于管理和查看来自 Android 端上传的截图。暂不接入 AI 分析，先跑通基础流程。

---

## 二、核心需求（必须实现）

### 1. 首页 - 实时上传展示
- 实时展示最新上传的截图
- 列表形式展示，包含：缩略图、上传时间
- 点击查看大图

### 2. 历史记录页
- 分页展示所有历史记录
- 支持按日期范围筛选
- 支持删除单条记录

---

## 三、详细功能拆解

### 1. 后端 API（复用现有）
- 保持 `/upload` 接口不变（接收上传）
- 新增 `/api/records` - 获取记录列表（分页）
- 新增 `/api/records/:id` - 获取单条记录详情
- 新增 `/api/records/:id/delete` - 删除记录

### 2. 前端页面
- **首页（Dashboard）**：最新记录列表 + 快速预览
- **历史记录（History）**：表格 + 筛选 + 删除

### 3. 数据存储
- 图片保存到 `backend/screenshots/`
- 记录元数据（时间戳、文件名）保存到 `backend/records.json`

---

## 四、技术选型

- 前端：React + Vite
- 后端：Node.js + Express
- UI 组件库：Ant Design

---

## 五、交付要求

1. 完整的前端项目代码
2. 后端 API 完善
3. 本地可以直接运行访问
4. Android 上传后网站立即显示
